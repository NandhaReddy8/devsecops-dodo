=== NetworkPolicy defence-in-depth verification (kind default CNI = kindnet) ===

1. Legitimate traffic (reporting -> ledger-api, covered by allow-ledger-api-ingress):
$ kubectl -n payments exec deploy/reporting -c client -- curl -s --max-time 5 -o /dev/null -w "HTTP %{http_code}\n" http://ledger-api:8080/health
HTTP 200

2. Unauthorized pod (netpol-test, no matching allow rule, default-deny-all applies):
$ kubectl -n payments exec netpol-test -- curl -s --max-time 5 -o /dev/null -w "HTTP %{http_code}\n" http://ledger-api:8080/health
(curl exit code 28: Operation timed out) HTTP 000 - no response at all.

Confirms kindnet enforces NetworkPolicy in this cluster: the unauthorized
pod is blocked at L3/L4 before the request ever reaches the Istio sidecar
(compare to the Istio AuthorizationPolicy test in
task3-authz-allow-deny.txt, which gets an HTTP 403 response FROM Envoy -
a strictly later, L7, identity-aware rejection). This is the concrete
difference between the two layers referenced in task3-mesh/README.md.
